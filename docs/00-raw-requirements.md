# 原始需求文档

## 来源
用户（绪星/midiaxy）在 Telegram 群组提出：
> "https://github.com/ArjanCodes/examples/tree/main/2025/logic 我有个想法，要你来和我讨论一下。我觉得这个目录的内容很适合新手学习，我想基于这里的内容设计一个 ai 指导新手学习的课程，ai 一步步引导新手优化代码，最后总结对话，给于评分。"

## 核心目标

1. **内容基础**：基于 ArjanCodes/examples/2025/logic 目录的代码重构教学内容
2. **目标用户**：编程新手
3. **教学模式**：AI 一步步引导
4. **交互终点**：代码优化 + 对话总结 + 评分

## 产品形态（用户进一步明确）

> "产品形态就是一个 git ，里面包含 https://github.com/ArjanCodes/examples 的内容。用户检出 git 项目后，在这个目录启动 opencode，ai 询问要学习的目录，然后就指导用户学习。"

**关键要素**：
- Git 仓库形式分发
- 包含 ArjanCodes 原始内容
- 在目录中启动 opencode
- AI 主动询问学习目录
- 指导用户学习

## 代码修改方式（用户提出）

> "用户修改代码的方式可以考虑：
> - vscode 等编辑器手动改
> - 用户用自然语言告诉 ai
> - ai 提供修改的 diff（多个选项）让用户选择
> - 用户修改好后，ai 来审核指导"

**三种模式**：
1. 手动编辑（VS Code 等）
2. 自然语言指令（告诉 AI 怎么改）
3. AI 提供多选项 diff 让用户选

## 补充想法（讨论中提出）

> "是否可以设计成我在 telegram 上可以和你对话来学习？"

**新增渠道**：Telegram 对话式学习

## 需求边界（讨论中明确不做）

- 不追求复杂的评分算法（前期不重要）
- 不是多人在线协作
- 不播放视频内容
- 不是 IDE 插件（可能后期做，但非 MVP）
- 不自动批改代码（AI 指导为主）

## 成功标准（讨论中定义）

1. 用户可以从头到尾完成 logic 课程的学习
2. 用户通过自然语言与 AI 互动
3. 每课结束后，用户能清晰看到自己的进步
