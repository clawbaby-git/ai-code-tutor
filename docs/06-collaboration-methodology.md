# AI 协作方法论：分层渐进式讨论

## 核心原则

**自底向上、逐层锁定、避免过早深入细节**

## 四层讨论框架

```
Layer 4: 实现层 (How to build)
Layer 3: 交互层 (How to interact)
Layer 2: 结构层 (What to build)
Layer 1: 价值层 (Why build)
```

**关键规则**：下层未定，不深入上层；每层讨论完毕，文档锁定。

## 每层讨论的标准输出

| 层级 | 讨论目标 | 标准输出文档 |
|-----|---------|------------|
| L1 价值 | 确认目标用户、核心痛点、成功标准 | `01-product-vision.md` |
| L2 结构 | 确定数据模型、目录结构、组织方式 | `02-content-structure.md` |
| L3 交互 | 确定用户操作流程、对话模式 | `03-interaction-design.md` |
| L4 实现 | 确定技术栈、MVP范围、开发计划 | `04-implementation-plan.md` |

## 讨论节奏控制

### 每层内部：先发散再收敛

```
发散阶段 -> 对比阶段 -> 收敛阶段 -> 锁定阶段
```

### 关键决策点标记

讨论中遇到需要用户拍板的决策，明确标记：

> 【决策点】选项 A vs 选项 B，你的选择是？

## 避免常见陷阱

| 陷阱 | 症状 | 解法 |
|-----|------|------|
| 过早优化 | L2 还没定就讨论实现细节 | 强制分层 |
| 方案膨胀 | 每个决策点给出太多选项 | 每层最多 3 个选项 |
| 文档滞后 | 讨论很多但无文档锁定 | 每层结束必须输出文档 |
| 用户失焦 | 讨论偏离原始需求 | 定期回顾 `00-raw-requirements.md` |

## 快速启动模板

```markdown
# 项目名称讨论记录

## 原始需求
（用户最初的话，原封不动粘贴）

## 讨论分层进度
- [ ] L1 价值层
- [ ] L2 结构层
- [ ] L3 交互层
- [ ] L4 实现层
```

## 应用实例

本方法论应用于 `ai-code-tutor` 项目的讨论过程。
